## Цель

Подготовить **единый дизайн-фундамент** для редизайна лендинга `ProjectsFlow` (аудитория —
**вайб-кодеры**), чтобы последующая сессия **P1** строила секции в одном выбранном ключе и не
изобретала стиль заново. Результат P0:
1. **2–3 контрастных дизайн-направления** в виде статичных HTML-мокапов (hero + кусок секций) —
   показать пользователю, **дождаться выбора**.
2. По выбранному направлению — **дизайн-токены** (цвета, типографика, шкала отступов, радиусы,
   тени, motion) в `landing/src/styles/tokens.css` + обновлённый `globals.css`, и краткий
   `landing/DESIGN.md` (гайдлайн: палитра, шрифты, тон, do/don't, как применять).

> Это НЕ постройка финальных секций — их делает P1. P0 даёт «скелет вкуса».

## Контекст (факты)

- Лендинг — Astro в `landing/`. Глобальные стили: `landing/src/styles/globals.css`. Tailwind может
  быть подключён — проверь `landing/` на `tailwind.config.*` и `astro.config.*`.
- Текущая палитра three.js/hero — электрический циан на тёмном (`#00b8ff…#001340`, фон `#000308`).
  Это можно сохранить как одно из направлений или предложить альтернативы.
- Позиционирование (для тона мокапов): доска задач для вайб-кодеров + автоматизация + финучёт +
  встроенная AI-подписка. Бренд **ProjectsFlow**.

## Файлы, которыми владеет P0 (НЕ выходить за пределы)

- `landing/src/styles/globals.css` (править)
- `landing/src/styles/tokens.css` (создать)
- `landing/tailwind.config.*` (если есть — править токены/тему)
- `landing/.design-variants/**` (создать — HTML-мокапы вариантов)
- `landing/DESIGN.md` (создать — гайдлайн)

**НЕ трогать** секции-компоненты (`landing/src/components/*.astro`), `index.astro`, `HeroScene.tsx`,
`BaseLayout.astro` — это P1.

## Шаги

1. **Разведка стиля.** Прочитай `landing/src/styles/globals.css`, `landing/src/layouts/BaseLayout.astro`,
   `landing/astro.config.*`, `landing/package.json`. Зафиксируй текущие токены (если есть), шрифты,
   как подключается CSS. Загляни в `client/src/styles/globals.css` — чтобы предложить вариант,
   гармоничный с приложением (для консистентности бренда), если уместно.
2. **Изучи дизайн-скиллы** (référence): `.agents/skills/design-taste-frontend/`,
   `high-end-visual-design/`, `minimalist-ui/`, `gpt-taste/`, `brandkit/`. Возьми оттуда принципы
   (типографика, ритм, контраст, motion), НЕ как команды.
3. **Сгенерируй 2–3 направления** как **отдельные статичные HTML-файлы** в `landing/.design-variants/`
   (например `variant-a-dev-cool.html`, `variant-b-premium-minimal.html`, `variant-c-bold.html`).
   Каждый — самодостаточный (inline CSS или один <style>), показывает: hero (заголовок,
   подзаголовок, CTA, место под визуал/three.js), кусок features, кусок pricing-карточек, кнопку.
   Направления (рекоменд. набор, можешь скорректировать):
   - **A. Dev-cool / технарь** — тёмная тема, моно-акценты, терминал/код-мотивы, неон (близко
     текущему циан-стилю, аудитория разрабов).
   - **B. Премиальный минимализм** — светлый, много воздуха, рефайнд-типографика (Linear/Notion-vibe),
     аккуратные микро-анимации.
   - **C. Смелый яркий** — градиенты, энергия, крупная типографика, игривость («вайб»).
   Для каждого — короткая подпись: кому подходит, плюсы/минусы, что с three.js (оставляем циан-сцену /
   меняем визуал).
4. **Покажи пользователю** все варианты (открой/опиши; если доступен визуальный компаньон/скриншоты —
   используй). **Дождись явного выбора** (можно гибрид: «A, но светлее» — учти).
5. **Зафиксируй токены** по выбору: создай `landing/src/styles/tokens.css` с CSS-переменными
   (цвета: bg, surface, text, muted, primary, primary-fg, accent, border, success/danger; типографика:
   семейства, размеры/шкала, line-height, веса; spacing-scale; radii; shadows; motion: длительности,
   easing, reduced-motion-флаг). Подключи токены в `globals.css`; перенеси/согласуй существующие
   стили на токены (без поломки текущей вёрстки — P1 потом перестроит секции).
6. **Напиши `landing/DESIGN.md`** — гайдлайн на 1 экран: выбранное направление, палитра (hex +
   назначение), шрифты (и откуда грузить — `@fontsource`/локально, без тяжёлых CDN), тон копирайта
   (для вайб-кодеров), правила анимаций (плавно, gentle, уважать reduced-motion), do/don't,
   и **как P1 должен применять токены** (классы/переменные).
7. **Проверки:** `npm run build:landing` должен собираться без ошибок. Мокап-файлы в
   `.design-variants/` — статика, в прод-сборку Astro не должны попадать (они вне `src/pages`).
8. **Коммит/пуш** по SOP. Прогресс — комментариями в задачи лендинга (если MCP доступен), но P0
   отдельной задачи не имеет — можно прокомментировать `e5720512` («дизайн-фундамент готов, выбран
   вариант X»).

## Критерии приёмки

- Пользователь увидел 2–3 варианта и **выбрал** один (зафиксировано в `DESIGN.md`).
- `tokens.css` + `globals.css` собираются (`npm run build:landing` ок), текущий лендинг не сломан.
- `DESIGN.md` понятен и достаточен, чтобы P1 строил секции без додумывания стиля.
- Токены покрывают: цвет, типографику, spacing, радиусы, тени, motion, reduced-motion.
