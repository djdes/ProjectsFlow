## 1. Цель и принципы

Превратить лендинг ProjectsFlow в **продающий лендинг для вайб-кодеров** в выбранном в P0
направлении. Принципы:
- **Один нарратив:** «бросаешь задачу → AI-воркер пишет код по нашей подписке; автоматизация и
  финучёт работают сами». Каждая секция усиливает этот нарратив.
- **Grand Offer (Hormozi):** value stacking + сильная гарантия + снятие рисков. Использовать скилл
  `grand-slam-offer` (доступен в Skill-туле) — но обещать только то, что реально есть.
- **Дизайн-консистентность:** строго по токенам/гайдлайну P0 (`landing/DESIGN.md`,
  `landing/src/styles/tokens.css`). Никакого «второго стиля».
- **Производительность и доступность:** лёгкий бандл, lazy-загрузка тяжёлого, `prefers-reduced-motion`,
  адаптив 320px→desktop, семантический HTML, корректные мета-теги.
- **Честность:** копирайт без вранья; гарантия — реальная.

## 4. Контент-контракт (общий с мастер-планом — НЕ менять числа произвольно)

**Бренд:** ProjectsFlow. **Аудитория:** вайб-кодеры (инди-разработчики, делают проекты с AI).
**Тон:** живой, по делу, без корпоратива; уважительный к разрабам.

**Ключевые ценности (для Hero/Features):**
1. Доска задач + колонка **worker** — бросаешь задачу, AI пишет код/коммитит/деплоит.
2. **Автоматизация** — диспетчер сам генерит и выполняет задачи по критериям, до лимита.
3. **Финучёт по проекту** — расход на AI/API по каждому проекту, в рублях, прозрачно.
4. **Встроенная AI-подписка** — без своих ключей, один тариф на все проекты.
5. (доп.) GitHub-привязка, коммиты↔задачи, KB, секреты.

**Тарифы (витрина, `landing/src/data/pricing.ts`):**
```ts
export const PLANS = [
  { id:'free',  name:'Бесплатный', priceRub:0,    priceNote:'навсегда',
    features:['до 5 проектов','колонка worker','задачи по твоей подписке','доска + коммиты'],
    cta:'Начать бесплатно', highlight:false },
  { id:'prime', name:'Прайм', priceRub:1900, priceUsd:20, priceNote:'в месяц',
    features:['проекты на наших подписках','самостоятельно, без нашей помощи','автоматизация + автодеплой','можно попробовать 1 час'],
    cta:'Перейти на Прайм', highlight:true },
  { id:'vip',   name:'ВИП', priceRub:3900, priceNote:'в месяц',
    features:['настройка проекта с нами','подробная инструкция','приоритетная поддержка'],
    cta:'По запросу', highlight:false, requestOnly:true },
] as const;
```
> Это **витрина** — без реальной логики подписки (см. мастер §1). Если пользователь захочет
> поправить цены/формулировки — менять ТОЛЬКО здесь.

**Grand Offer / гарантии (для `Guarantees.astro`):** гарантия возврата денег (если не подошло —
вернём), прозрачность списаний (видишь каждый рубль), нет лок-ина (данные/код в твоём GitHub),
быстрый старт (бесплатный тариф без карты). Сформулировать как «оффер, от которого сложно
отказаться», value-stack рядом с ценой.

### B. Позиционирование (`e5720512`)

   направления как образец копи). CTA: «Создать аккаунт» + «Войти» через `AuthCta.astro`
   (сохранить auth-aware). Eyebrow «для вайб-кодеров».
4. **Features (`Features.astro`)**: 4–5 карточек по ценностям §4. Иконки + заголовок + 1–2 строки.

5. **HowItWorks (`HowItWorks.astro`)**: 3–4 шага под новый нарратив (заведи проект → подключи
   GitHub → бросай задачи воркеру → автоматизация и финучёт сами). Сохрани/обнови step-connector.

### C. Новые секции (порядок в `index.astro`)
6. Итоговый порядок: `Nav → Hero → Features → HowItWorks → Screenshots → Pricing → Guarantees →
   Faq → CTASection → Footer` (дивайдеры между). Скорректируй по эстетике направления.

### D. Скриншоты/примеры (`9ae0842f` → `Screenshots.astro`)
7. Секция с реальными скринами продукта (доска, окно задачи, окно автоматизации, финансы). Источник
   изображений: **спросить пользователя** или сделать качественные плейсхолдеры (рамка-браузер +
   подпись), пометить TODO «заменить на реальные». Картинки — `loading="lazy"`, `width/height`
   заданы (без CLS), responsive `srcset` при возможности.

### E. Grand Offer / гарантии (`9ae0842f` → `Guarantees.astro`)
8. Применить `grand-slam-offer`: value-stack (что получаешь) + сильная гарантия возврата + снятие
   рисков (нет лок-ина, прозрачность, без карты). Визуально — карточки/бейджи гарантий рядом с
   ценами или отдельной полосой перед/после Pricing.

### F. Витрина тарифов (`0c770033` + `903d484f` → `Pricing.astro`)
9. 3 карточки из `pricing.ts`: фичи списком, выделенный **Прайм** (`highlight`), у **ВИП** —
   «по запросу» (не самоподключение), у **Прайм** — «попробовать 1 час» (текст витрины). Адаптив:
   3 кол → 1 кол на мобиле. Связать с гарантией (рядом — «гарантия возврата»).

### G. three.js — фикс (просьба пользователя)
10. В `Hero.astro`/`HeroScene.tsx`:
    - **reduced-motion:** если `matchMedia('(prefers-reduced-motion: reduce)')` — НЕ монтировать
      сцену; показать статичный кадр (poster `.webp`/`.avif` или CSS-градиент в стиле сцены).
    - **mobile-off:** не грузить и не монтировать `HeroScene` при ширине < 1024px (и/или
      coarse-pointer). В Astro — обернуть монтаж условием на клиенте (island грузит chunk только
      при выполнении условия) либо `client:media="(min-width: 1024px)"`; на мобиле — статичный
      poster, чтобы chunk three.js (~0.5 МБ) НЕ скачивался.
    - **fallback на слабых GPU / ошибке WebGL:** `try/catch` инициализации + проверка
      `WebGLRenderingContext`; при провале — тот же статичный poster. Сохранить AdaptiveDpr/instancing.
    - **перф-бюджет:** убедиться, что на мобиле LCP не блокируется сценой; poster лёгкий (<40 КБ).

### H. Сквозной редизайн (дизайн-скиллы)

12. Подключить **чат-виджет** (P2) в `BaseLayout.astro` (один импорт острова снизу справа) — если
    P2 готов; иначе плейсхолдер с TODO.

### I. Мета / SEO / соц-карточки
13. В `BaseLayout.astro`: `<title>`, `<meta name="description">` (про доску задач для вайб-кодеров),
    OpenGraph/Twitter (`og:title/description/image`), `lang="ru"`, canonical. OG-картинка — в
    `landing/public/og.png`.

## 6. Адаптив / доступность / перф (обязательно)

- **Брейкпоинты:** проверить 320 / 375 / 414 / 768 / 1024 / 1280+. Ничего не должно ломаться/
  скроллиться горизонтально.
- **A11y:** семантические теги (`header/nav/main/section/footer/h1..h3`), alt у картинок,
  контраст текста по выбранной теме (WCAG AA), фокус-стейты у кнопок/ссылок, кликабельные области
  ≥ 40px на тач.
- **Reduced-motion:** все анимации (включая three.js) уважают `prefers-reduced-motion`.
- **Перф:** three.js не грузится на мобиле/reduced-motion; картинки lazy + размеры заданы; шрифты
  через `@fontsource`/локально (без тяжёлых CDN); `npm run build:landing` без варнингов о размере,
  где возможно.

## 8. Риски / edge-кейсы

- **Astro island + three.js на мобиле:** убедиться, что `client:media`/условный монтаж реально не
  тянет chunk three.js на мобиле (проверить Network в DevTools). Это главный перф-выигрыш.
- **Auth-aware CTA:** не сломать проверку `/api/auth/me` (залогиненный видит «Открыть приложение»).
- **Конфликт с P0:** не править `globals.css`/`tokens.css`. Новые стили — в компонентах/scoped.
- **Зависимость от P2:** если виджет-острова ещё нет — плейсхолдер, не блокироваться.
- **Реальные скриншоты:** если пользователь не дал — качественные плейсхолдеры + TODO, не выдумывать
  фейковые «отзывы»/логотипы клиентов (честность).
