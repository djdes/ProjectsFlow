# Дизайн: редизайн страницы «Обзор» проекта + кнопка «Создать репо»

Дата: 2026-07-12. Статус: утверждён, ждёт реализации (отложена, пока параллельная
сессия работает со страницей проекта).

## Цель

Привести страницу «Обзор» проекта (`client/src/presentation/pages/ProjectPage.tsx`)
к общему стилю приложения (широкая, плоская, Notion-подобная — как «Входящие» и доска)
и добавить возможность создать новый GitHub-репозиторий прямо из проекта с
автозаполнением имени по названию проекта.

## Контекст / что уже есть

- Бэкенд умеет создавать репо: `GithubApiClient.createRepo()` — используется KB-репо
  и app-репо воркера (`server/src/application/project/EnsureProjectAppRepo.ts`).
  Там же транслит-`slugify` («Обувь» → `obuv`) — сейчас приватный, надо вынести.
- `POST /api/projects/:id/app-repo` — флоу воркера (имя `pf-<slug>-<id8>`, включает
  делегацию токена, заводит KB, кладёт workflow). **Не переиспользуем** для
  пользовательской кнопки — у неё нет побочных эффектов.
- PATCH `/api/projects/:id` уже принимает `status: active|paused|archived`
  (`server/src/presentation/projects/schemas.ts:51`) — переключатель статуса не
  требует бэкенда.
- Право `update_project` = editor+ (`server/src/domain/project/permissions.ts`).
- Эталон отступов: `InboxPage.tsx` — тело `px-6 sm:px-14 lg:px-24`, крошки `h-11 px-2.5`.

## Критика текущей страницы (зафиксировано)

1. Единственная страница с узкой центрированной колонкой (`max-w-3xl mx-auto`) из
   жирных shadcn-карточек — выбивается из стиля Входящих/доски.
2. Смешаны «обзор» (коммиты, команда) и «настройки» (репо, KB, диспетчер, опасная
   зона); полное разделение — отдельный будущий этап, сейчас только задел порядком секций.
3. Пустые состояния — громкие onboarding-баннеры с большими синими кнопками.
4. Кнопка «Доска задач» дублирует навигацию; бейдж «Активен» мёртвый (не кликается);
   «Мои уведомления» (личная настройка) живёт внутри секции «Команда».

## Решение (вариант C: косметика + задел под разделение)

### 1. Каркас страницы

- Крошки — без изменений (sticky, h-11, px-2.5).
- Тело: `px-6 sm:px-14 lg:px-24` (как Входящие/доска), контент прижат **влево**
  (убрать `mx-auto`), текстовым секциям — `max-w-3xl` для читаемости строк.
- Под `EditableProjectTitle` — одна мета-строка: **переключатель статуса**
  (dropdown: Активен / На паузе / Архив; PATCH `{status}`) + тихая ghost-ссылка
  «Доска задач» (оставляем — на мобиле это главный переход).

### 2. Секции: карточки → плоские блоки

Единый паттерн: иконка + заголовок `text-sm font-medium`, тонкий разделитель между
секциями, без Card-рамок. Пустые состояния — одна muted-строка с маленькими кнопками,
не баннер.

Порядок (живое → настроечное → личное → опасное):

1. Заголовок + статус + доска
2. Команда
3. GitHub-репозиторий, сразу под ним «Последние коммиты»
4. База знаний
5. Диспетчер
6. Мои уведомления — **вынести из `TeamSection`** в собственный блок
7. Опасная зона — одна строка с ghost-кнопкой «Удалить проект» (диалог остаётся)

Внутреннюю логику `TeamSection`/`KbSection`/`DispatcherSection`/`RecentCommitsSection`
не трогаем — только обёртки и стили (минимизация конфликта с параллельной сессией).

### 3. Кнопка «Создать репо»

Клиент:

- Empty-state GitHub-секции: `[Выбрать из GitHub] [Создать новый]`.
- В футере `RepoPickerDialog` — та же ссылка «Создать новый».
- Новый `CreateRepoDialog`: имя (предзаполнено транслит-slug'ом названия проекта,
  редактируемое, валидация `[a-z0-9._-]`), toggle «Приватный» (default on),
  кнопка «Создать». 422 «имя занято» — inline-ошибка в диалоге, без авто-суффиксов.
- Видна только при подключённом GitHub текущего юзера и роли editor+.

Сервер:

- Use-case `CreateProjectRepo`: право `update_project`; репо создаётся токеном
  **текущего юзера** (не владельца проекта); `createRepo({autoInit: true, privateRepo})`;
  затем `projects.update({gitRepoUrl})`. Отказ (конфликт), если `gitRepoUrl` уже задан.
- Маршрут `POST /api/projects/:id/repo`.
- `slugify` вынести из `EnsureProjectAppRepo` в общий модуль, использовать в обоих местах.
- Без побочных эффектов app-repo-флоу (делегация, KB, workflow).

### 4. Ошибки и краевые случаи

- Нет GitHub-коннекта → как сейчас: «Подключить GitHub в профиле».
- 422 имя занято → inline-ошибка, юзер правит имя.
- Токен протух/отозван → toast «переподключи GitHub в профиле».
- Мобила: те же секции, `px-6`, мета-строка переносится; спец-логики не нужно.
