# «Входящие»: подтверждение делегирования + фильтр по клику на аватар

Дата: 2026-07-13. Статус: дизайн утверждён юзером. Клиентское изменение, один компонент
`client/src/presentation/components/tasks/AssignedToMeBlock.tsx` (+ under-компонент `UserCube`
в том же файле). Серверных изменений нет.

## Контекст

Блок «Входящие» (`AssignedToMeBlock`) — две вкладки: «Для меня» (`toMe`) и «Другим» (`byMe`).
Ряд «кубиков» участников пространства (`UserCube`, из `listSharedMembers`) — сейчас это только
drop-цель делегирования (перетащил задачу на кубик → `reassignTo`). Вкладка «Другим» уже имеет
single-select фильтры «от кого / кому / проект» (`filterFrom/filterTo/filterProject`),
портализованные в шапку страницы; предикат `matchesByMeFilters`.

## Фича 1 — подтверждение делегирования при drop на аватар

Сейчас `handleDragEnd`, ветка `data.type==='user'` не-своего кубика, сразу зовёт
`reassignTo(item, member)`. Заменить на: открыть модалку подтверждения.

- Стейт `pendingReassign: { item: AssignedTask; member: SharedMember } | null`.
- В `handleDragEnd` для делегирования на другого человека — `setPendingReassign({item, member})`
  вместо немедленного `reassignTo`. Дроп на свой кубик (`reclaimToSelf`) и на колонки-сроки —
  БЕЗ подтверждения (не делегирование).
- Модалка (`Dialog`, уже импортирован): заголовок «Делегировать задачу?»; тело — название задачи
  (plain, первая строка `splitDescription().name` или существующий способ показа заголовка),
  строка «Сейчас у: <`item.delegation.delegateDisplayName`>» и «Делегировать: <`member.displayName`>».
  Кнопки «Делегировать» (primary) → `reassignTo(item, member)` затем закрыть; «Отмена» → закрыть
  без действия. Закрытие по фону/Esc = Отмена.

## Фича 2 — клик по аватару = single-фильтр «кому» (только вкладка «Другим»)

Кубик `UserCube` — `useDroppable` (drop-цель), НЕ draggable, поэтому клик по нему это обычный
`onClick`, отдельный от drag (задачу тащат, кубик кликают) — disambiguation не нужна.

- Новые пропы `UserCube`: `filterActive?: boolean`, `onToggleFilter?: () => void`.
- Рендер ряда кубиков передаёт эти пропы ТОЛЬКО на вкладке «Другим» (`tab==='byMe'`); на «Для меня»
  пропы не передаются → кубик как сейчас (только drop-цель, без клик-фильтра и без X).
- `onToggleFilter` = `setFilterTo(prev => prev === member.id ? null : member.id)` — клик по
  неактивному ставит фильтр на него; клик по активному снимает; клик по другому переключает.
- `filterActive = tab==='byMe' && filterTo === member.id`. Активный кубик визуально выделен
  (напр. `bg-primary/10 ring-1 ring-primary/30`) и показывает маленький крестик (× в правом углу,
  `pointer-events` кликабельный): клик по × — `stopPropagation` + `setFilterTo(null)` (снять).
- Клик по телу кубика (не по ×) вызывает `onToggleFilter`. Во время drag (`dragging`) клик-фильтр
  не срабатывает (drag-режим показывает ринги делегирования) — либо через `dragging` пропа гасить
  onClick, либо onClick и так не сработает при drag (dnd тащит карточку, не кубик).
- **Синк с шапочным фильтром — автоматический:** `filterTo` — общий стейт. Выставил «кому» через
  меню-фильтр в шапке → `filterActive` у соответствующего кубика становится true → появляется ×.
  Снял — × исчезает. Обратно: клик по кубику меняет тот же `filterTo`, который читает шапочный
  фильтр-чип. Ничего дополнительно синкать не нужно.
