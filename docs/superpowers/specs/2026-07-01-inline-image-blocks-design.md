# Блоки-картинки в описании задачи (inline + подпись + drag-reorder)

> Дизайн одобрен владельцем 2026-07-01 (вариант A, подтверждён скрином). Редактор — TipTap 3
> (`RichTextEditor`, хранение = markdown). Цель: вставлять скрины Ctrl+V как блок в позицию,
> подписывать снизу, переставлять блоки ручкой-«6 точек», и чтобы воркер понимал позицию скрина.

## Что делаем

1. **Вставка скрина** (Ctrl+V / paste image) в окне создания И редактирования задачи → картинка
   грузится как вложение задачи и вставляется **блоком в позицию курсора** (не в отдельный список).
   Не-картинки — прежний путь (отдельные вложения через `onPasteFiles`).
2. **Блок-картинка = «фигура»**: `<figure>` = картинка + редактируемая **подпись снизу**
   (placeholder «добавить подпись»).
3. **Ручка-«6 точек»** слева при наведении на любой блок (абзац/картинка) → drag-reorder блоков.
4. **Прогресс загрузки**: пока скрин грузится — на его месте блок-плейсхолдер (иконка картинки +
   горизонтальный прогресс-бар, движется по `uploadAttachment (loaded,total)`), затем блок
   превращается в саму картинку. Ошибка → блок убирается + toast.
5. **Воркер понимает позицию** скрина: см. «Формат».

Нода `figureImage` держит состояния через атрибуты: `uploading`(bool)+`progress`(0-100) → плейсхолдер
с баром; после загрузки `src` задан → картинка+подпись. В markdown сериализуются только загруженные
(`uploading=false`); загрузка — транзиентное UI-состояние.

## Формат (ключевое) — вариант A

- Описание остаётся **markdown**. Картинка встраивается **inline в свою позицию**:
  `![подпись](url-вложения)`. В тексте буквально: `абзац` → `![подпись](скрин)` → `абзац`.
- Воркер, читая описание (`pf_get_task`), **видит порядок**: после какого абзаца какой скрин и его
  подпись (alt). Сам бинарь скрина он открывает как вложение (pf_get_task отдаёт картинки).
- **Бэкенд/БД НЕ меняем** — вложения уже хранятся, позиция живёт в markdown-описании.

## Реализация

### Клиент (`client/`)
- **Нода `figureImage`** (`editor/extensions/`): NodeView рендерит `<figure><img/><figcaption/></figure>`;
  атрибуты `src`, `alt`(=подпись). `parseHTML`/`renderHTML` для round-trip;
  `renderMarkdown(node) → ![alt](src)` (как `TextStyle.renderMarkdown` уже сделан в `buildExtensions`);
  парс markdown-картинки `![]()` при `setContent({contentType:'markdown'})` — через `@tiptap/markdown`
  (нода `image`/кастомный parseMarkdown). Подпись редактируется inline, placeholder «добавить подпись».
- **Drag-handle**: `@tiptap/extension-drag-handle-react` (в семействе `@tiptap`, мелкая доп-зависимость) —
  ручка-«6 точек» слева на hover, включает перетаскивание блоков. Стиль — 2×3 точки (как в примере).
- **Paste → inline**: в `RichTextEditor.handlePaste` для image-файлов: загрузить вложение
  (`taskRepository.uploadAttachment` → вернуть url/id) и `editor.commands.insertContent` ноды
  `figureImage` с `src`. Прокинуть загрузчик в редактор (проп `onPasteImage(file) → Promise<{url}>`),
  чтобы редактор не знал про репозиторий напрямую (boundaries). Не-image → прежний `onPasteFiles`.
  Точки монтажа: `TaskDrawer` (create/edit) и `AddTaskDialog`.
- **Read-вью** (`markdown/Markdown.tsx`): разрешить `img`/`figure`/`figcaption` в SANITIZE_SCHEMA и
  отрендерить `![alt](url)` как фигуру с подписью (симметрично редактору).

## Критерии приёмки
- Ctrl+V скрина в создании и редактировании задачи → картинка блоком в позиции курсора + подпись снизу.
- Подпись редактируется и сохраняется (в alt). Drag-«6 точек» переставляет абзацы и картинки.
- В сохранённом описании (markdown) картинка стоит `![подпись](url)` ровно в своей позиции; воркер
  через `pf_get_task` видит позицию и подпись.
- `npm run typecheck` + `lint` чисто; read-вью рендерит фигуру.
