# Ежедневная сводка в Telegram: жирные заголовки, кнопки задачи, без «Готово», сплит

Дата: 2026-06-09
Статус: реализован

## Требования (от пользователя)

1. Заголовок задачи — **жирный** (и «крупнее»), **не гиперссылка**.
2. В конце каждой задачи — ссылки: **«Комментировать (N) | Завершить»**.
   - «Комментировать» → сайт, карточка задачи (комментарии/правка); `(N)` = актуальное число
     комментариев задачи.
   - «Завершить» → как сейчас: перевод задачи в «Готово» + уведомления по правилам проекта.
3. Убрать в конце сообщения секцию **«Готово»** (уже выполненные задачи).
4. Длинная сводка → **несколько сообщений** (все задачи целиком, без обрезки). В сводке не
   показываем комментарии (только счётчик).

## Ограничение платформы

Telegram **не умеет размер шрифта** — «крупнее» буквально нельзя. Решение (согласовано):
заголовок `<b>…</b>` на отдельной строке + пустая строка-разделитель между задачами.

## Что было

`buildTaskDigest.ts` → `renderDigestTelegram(model)` возвращал ОДНУ строку, обрезал на ~3800
символах с «…полностью на сайте». Заголовок — гиперссылка (`<a href="?task=">name</a>`), рядом
ссылка «✓ Готово» (`?task=&done=1`). Группировка по статусам включала `done`. `commentCount`
захардкожен `0`.

## Что стало (только Telegram-формат; email/markdown — лишь фильтр done)

- **Заголовок:** `<b>{name}</b>` (жирный, не ссылка), без нумерации «1.».
- **Мета:** `👤 {исполнитель} · ⏰ {срок}` (исполнитель теперь всегда в мете; опция
  `assigneeFirst` удалена — личка и группа идентичны).
- **Футер задачи:** `<a href="{openLink}">Комментировать ({N})</a> | <a href="{doneLink}">Завершить</a>`.
  При `N=0` — просто «Комментировать» (без «(0)»). `openLink=?task=<id>`, `doneLink=?task=<id>&done=1`.
- **Сплит:** `renderDigestTelegram` возвращает **`string[]`** — пакует блоки-задачи целиком в
  сообщения ≤ ~3800 символов; заголовок проекта на первом, «…(продолжение)» на следующих,
  группа-заголовок повторяется в начале каждого сообщения с её задачами. Обрезка убрана; одна
  гигантская задача усекается ТОЛЬКО по телу (по сырому markdown → re-render, HTML остаётся
  сбалансированным).
- **Done убран везде** (TG + email + in-app): `SendDailyDigest` фильтрует `status !== 'done'`
  до сборки модели; `statuses` без `done` (пустая группа «Готово» не рендерится).
- **Счётчик комментариев:** `SendDailyDigest` тянет `comments.countsByTasks(ids)` → реальное `N`.
- **Отправка:** и личка (`telegram.execute`), и группа (`telegramClient.sendMessage`) шлют
  массив кусков по очереди.

## «Завершить» оповещает — проверено

`doneLink` (`?task=…&done=1`) → фронт (`KanbanBoard`) показывает подтверждение → POST move →
`targetStatus:'done'` → `notifier.onTaskDone(projectId, userId, task, 'team')`:
- **Email** участникам проекта (по их `notificationPrefs`, событие `task_done`, source `team` —
  по умолчанию всем, кроме инициатора);
- **Telegram-бродкаст** по задаче всем участникам (`BroadcastTelegramNotificationByTask`);
- **SSE** `task_status_changed` realtime.
То есть «как сейчас» + уведомления реально уходят. Меняем только подпись ссылки на «Завершить».

## Затронуто (всё server-only, без миграции)

- `server/src/application/task/digest/buildTaskDigest.ts` — `DigestItem.commentCount`,
  новый `renderDigestTelegram` (массив) + `digestItemBlockTg`.
- `server/src/application/digest/SendDailyDigest.ts` — фильтр done, счётчик комментариев,
  отправка массива.
- `server/src/application/task/ExportTasksDigest.ts` — адаптация ручного экспорта под массив.
- `server/src/index.ts` — DI: `comments: taskCommentRepo` в `SendDailyDigest`.
- тесты `buildTaskDigest.test.ts`.

## Вне рамок

Email/markdown-формат не меняем (только убрали done). One-tap inline-кнопки в Telegram
(невозможны в одном сообщении на множество задач — поэтому гиперссылки, как и просил пользователь).
Редактирование текста перефраза/комментариев в сводке (сводка — только заголовок+мета+тело+ссылки).
