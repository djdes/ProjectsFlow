# Ежедневная сводка в письме: раскладка как в Telegram (кнопки снизу, «Комментировать»)

Дата: 2026-06-30
Статус: проектирование

## Проблема (от пользователя)

В письме «Ежедневная сводка» у каждой задачи заголовок-ссылка, а рядом сбоку приклеена
зелёная ссылка `· ✓ Готово`. Это читается как непонятная «кнопка сбоку». Пользователь хочет,
чтобы письмо выглядело **как в Telegram**: действия задачи внизу карточки, и среди них —
«Комментировать».

Telegram-формат уже сделан (см. [spec 2026-06-09](2026-06-09-daily-digest-telegram-format-design.md)):
жирный заголовок (не ссылка) → мета → тело → футер «Комментировать (N) | Завершить». Письмо
от него отстало — правим только email-рендер.

## Что меняем

Только `renderDigestHtml` в
[buildTaskDigest.ts](../../../server/src/application/task/digest/buildTaskDigest.ts) — раскладку
одной карточки задачи. Модель (`DigestModel`/`DigestItem`), markdown-рендер и Telegram-рендер
не трогаем. Миграций нет, фронт не трогаем (ссылки `?task=` и `?task=&done=1` те же).

## Новая раскладка карточки задачи (email)

Карточка в рамке (`border:1px solid #e2e8f0; border-radius:10px; padding:12px 14px`),
зеркало Telegram-блока:

1. **Заголовок** — жирный, НЕ ссылка (как в TG). Нумерация «1.» убирается (в TG её нет).
2. **Мета** — `👤 {исполнитель} · ⏰ {срок}`, серый мелкий текст (только если есть).
3. **Тело** — `markdownToRich(body, 'email')` как сейчас.
4. **Вложения** — `📎 ссылки` как сейчас.
5. **Кнопки (внизу)** — две email-кнопки-плашки (`<a>` c inline-стилями,
   `display:inline-block`, скруглением и фоном — bulletproof для Gmail/Apple Mail):
   - **«💬 Комментировать (N)»** → `openLink` (`?task=<id>`). При `N=0` — без «(N)».
     Стиль: светло-синяя контурная плашка (`background:#eff6ff; color:#2563eb;
     border:1px solid #bfdbfe`).
   - **«✓ Завершить»** → `doneLink` (`?task=<id>&done=1`). Стиль: зелёная заливка
     (`background:#16a34a; color:#fff`).

Подписи кнопок и поведение ссылок 1:1 с Telegram-футером, чтобы каналы не расходились.

## Тесты

`buildTaskDigest.test.ts` → тест `renderDigestHtml`:
- больше нет `✓ Готово` приклеенного к заголовку (старый ассерт удаляется);
- заголовок жирный и НЕ обёрнут в `<a>`;
- внизу кнопка «Завершить» с `doneLink` и «Комментировать» с `openLink`;
- проверка `commentCount=0` → «Комментировать» без «(0)» (по аналогии с TG-тестом).
- экранирование (`A &amp; B &lt;x&gt;`) сохраняется.

## Вне рамок

Telegram/markdown-форматы, модель дайджеста, фронт, планировщик, каналы рассылки.
