## Реализация

### 1. Визуал / вёрстка
- Причесать сетку, отступы, типографику; единый стиль секций. Привести переключатели/радио/инпуты к
  актуальному стилю приложения (как в недавнем редизайне окна задачи: size-8 ghost-иконки, аккуратные
  свитчи). Адаптив + мобильные инсеты.

### 2. Визуальное разделение блоков
- Каждая «автоматизация» — отдельная карточка/панель с заголовком и тонким разделителем
  (диспетчер, мультизадачный воркер, критерии, лимит/пауза, git-автор, флаги, деплой,
  Telegram-группа, ежедневная сводка, авто-обработка по коммитам). Чёткая визуальная сегментация.

### 3. Логика мастер-переключателя
- Определить набор «автоматизаций», которыми управляет мастер. **Рекомендация:** мастер агрегирует
  включённость основных автоматизаций окна — как минимум диспетчер (генерация по критериям),
  мультизадачный воркер, ежедневная сводка, авто-обработка по коммитам.
- Правила: мастер ВКЛ, если включена хотя бы одна; мастер ВЫКЛ, если все выключены; **выключение
  мастера → выключить все**. Включение мастера — **уточнить у пользователя** желаемое поведение
  (включать все? только диспетчер? восстанавливать прошлый набор?). По умолчанию: включение мастера
  включает диспетчер (минимально), не трогая остальные, ИЛИ восстанавливает последнее ненулевое
  состояние — выбрать с пользователем.
- Реализовать как производное состояние (master = OR(individual)), а действие master-off — батчем
  по всем. Сохранять консистентно через существующие репозитории.

### 4. История group-id + имя группы (ежедневная сводка / Telegram-группа)
- **История ID групп:** показывать ранее введённые `telegram_group_chat_id`. Источник — distinct
  значения из `project_digest_settings` по проектам текущего пользователя (новый GET-эндпоинт в
  `digest/routes.ts`, напр. `GET /api/me/telegram-groups` → `[{ chatId, title }]`). UI — combobox/
  `datalist` с подсказками вместо «голого» text-инпута; ручной ввод тоже оставить.
- **Имя группы:** при вводе/выборе chat_id — попытаться **спарсить название** через Telegram
  `getChat` (бот) на сервере (новый эндпоинт, напр. `POST /api/projects/:id/telegram-group/resolve`
  → `{ title }`), сохранять в `telegram_group_title`. Если бот не в группе/нет прав — мягкий
  фоллбэк (показать только id, без падений). Не блокировать сохранение.
