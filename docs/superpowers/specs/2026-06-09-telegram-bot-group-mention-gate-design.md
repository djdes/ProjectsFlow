# Telegram-бот: в группах реагировать только на обращение к нему

Дата: 2026-06-09
Статус: реализован

## Проблема

Бот @ProjectsFlow_Bot, добавленный в групповой чат, реагирует на КАЖДОЕ сообщение (пытается
создать задачу / отвечает «не привязано к задаче» на любой reply). В группе это спам.

## Решение

Гейт в `HandleTelegramWebhook.execute` для текстовых сообщений из чатов типа `group`/
`supergroup`: сообщение обрабатывается, ТОЛЬКО если оно адресовано боту —
1. текст содержит упоминание `@<botUsername>` (регистронезависимо, по границе слова), ИЛИ
2. это reply на сообщение самого бота (`reply_to_message.from.is_bot === true`).

Иначе — `return` (бот молчит). Личка (`private`) — без изменений (любой текст = задача).
Callback-кнопки и inline-режим уже явные обращения — не гейтятся.

Дополнительно: перед маршрутизацией `@<botUsername>` вырезается из текста — чтобы упоминание
не попало в текст задачи и чтобы команды вида `/help@BotName` (так Telegram шлёт команды в
группах) распознавались как `/help`.

Реплай-на-бота оставлен в условии намеренно: иначе в группе сломался бы сценарий «ответить
reply на карточку задачи → комментарий».

## Реализация

- `server/src/application/telegram/HandleTelegramWebhook.ts` — гейт в начале обработки
  `update.message` (после извлечения `text`/`chatId`); `text` стал `let` (вырезаем упоминание).
- Используются только уже доступные поля: `msg.chat.type`, `msg.reply_to_message.from.is_bot`,
  `deps.botUsername`. Без изменений типов/схемы/БД.

## Тесты

`HandleTelegramWebhook.test.ts` (новый): группа без упоминания → игнор; группа `@bot текст` →
задача из очищенного текста (регистронезависимо, упоминание в середине); группа `/help@bot` →
справка; группа reply-на-бота → reply-ветка; группа reply-на-не-бота → игнор; личка → без
изменений; личка с упоминанием → упоминание не вырезается.

## Вне рамок

Опционально (вне кода): privacy mode у @BotFather (Telegram сам не шлёт лишнее в группах) —
комплемент, но код-гейт надёжнее и не зависит от настройки.
