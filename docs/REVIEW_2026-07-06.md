## 1. Безопасность (бэкенд)

### S1 — CRITICAL: эскалация viewer → dispatcher → GitHub-PAT владельца
`domain/project/permissions.ts:85` разрешает `set_project_dispatcher` с роли **viewer**.
Сценарий: viewer в проекте с включённой делегацией токена →
1. создаёт себе agent-токен (`POST /api/agent/tokens`, доступно любому);
2. назначает **сам себя** диспетчером (`SetProjectDispatcher.ts:36`, проверка проходит);
3. `GET /api/agent/projects/:id/git-token` (`GetDelegatedGitToken.ts:47`) отдаёт
   **plaintext OAuth-токен владельца** со scope `repo`.

**Фикс:** поднять `set_project_dispatcher` до `owner` (минимум `editor`); выдачу
git-token гейтить на «назначен owner/admin», а не на самоназначение. Добавить аудит-лог выдачи токена.

### S2 — CRITICAL: viewer читает все секреты/креды в plaintext
`application/secrets/GetSecret.ts:20` и `application/agent/GetAgentCredential.ts:36`
гейтят чтение на `read_project` (viewer), тогда как **запись** требует `editor`
(`PutSecret.ts:21`). Асимметрия: read-only участник вытягивает все пароли/API-ключи.

**Фикс:** поднять гейт чтения секретов/кред до `editor` (лучше `owner`).

### S3 — MAJOR: нет CSRF, поддомены хостят недоверенный контент
`presentation/http.ts` — ни `helmet`, ни CORS, ни CSRF (0 совпадений грепом). Cookie
`sameSite:'lax'` без `domain`. Деплой воркер-сайтов и публичные доски живут на
`<slug>.projectsflow.ru` — тот же registrable-домен → **same-site**. `Lax` шлёт
сессионную куку на кросс-сабдоменные запросы; `multipart/form-data` — «simple request»
без preflight (`POST .../avatar`, `.../attachments`). Вредоносный задеплоенный
сайт-сосед делает state-changing CSRF от имени жертвы.

**Фикс:** (а) вынести задеплоенные сайты/публичные доски на **отдельный
registrable-домен** (тогда они cross-site и Lax спасает); либо (б) проверять
`Origin`/`Sec-Fetch-Site: same-origin` на всех мутациях + double-submit CSRF-токен.
Плюс подключить `helmet` (CSP, `X-Frame-Options: DENY` для app-origin).

### S4 — MAJOR: agent-токены без scope; `/me` отдаёт GitHub-токен
`GetMyAccount.ts:47` (`GET /api/agent/me`) возвращает `github.accessToken` plaintext;
токен даёт полный доступ (включая `DELETE /api/agent/projects/:id`). Утечка одного
токена из конфига MCP = захват аккаунта + GitHub-токен scope `repo`.

**Фикс:** ввести scope у agent-токенов (per-project / read-only); убрать
`accessToken` из `/me`; git-токен — только через per-project `/git-token` с аудитом.

### S5 — MAJOR: нет шифрования секретов at-rest
`DrizzleSecretsRepository` пишет `value` как есть; `user_github_tokens.access_token`
plaintext; `SECRETS_MASTER_KEY` есть, но используется только как HMAC-fallback. Дамп
БД = все креды всех тенантов. (Известно: vault-шифрование сняли — см. память
`project_secrets_plaintext`.)

**Фикс:** envelope-encryption через `SECRETS_MASTER_KEY` + миграция. Отдельный эпик,
но требует recovery-flow, чтобы не повторить потерю master-ключа.

### MINOR (security)
- **s-m1** `apiRoutes.ts:709/721` `dispatch-allowed/:taskId|user` без проверки принадлежности → утечка `billedUserId`/плана чужих задач. Гейтить на «caller — диспетчер этой задачи/проекта».
- **s-m2** `presentation/kb/schemas.ts:11` регэкс `^[a-z0-9_./-]+\.md$` пропускает `../`. Реального FS-traversal нет (ключ в БД), но defense-in-depth сломан → отклонять сегменты `..` и ведущий `/`.
- **s-m3** `auth/routes.ts:110` нет rate-limit на `POST /register` → массовая регистрация. Повесить существующий `InMemoryRateLimiter` по IP.
- **s-m4** нет security-заголовков (helmet/CSP/HSTS/X-Frame-Options) → clickjacking app-origin.
- **s-m5** `gitRepoUrl` без валидации схемы → `javascript:`-ссылка исполнится по клику в админке. Whitelist `http/https/git/ssh`.

## 2. Корректность бэкенда и БД

### B1 — MAJOR: коллизии позиций задач в канбане
`MoveTask.computePosition` (`MoveTask.ts:131`) считает `(before + after) / 2` без
ребаланса. После ~50 перестановок между одними и теми же соседями float-точность
схлопывается → две задачи получают равный `position` → нестабильный порядок и
«прыжки» при refetch. Плюс операция read-modify-write без блокировки: два
одновременных move (человек + воркер) — last-write-wins, порядок теряется.

**Фикс:** периодический ребаланс колонки (перенумеровать целыми с шагом при
сближении соседей < ε) либо перейти на LexoRank/строковые ранги. Как минимум —
детектить схлопывание (`|before-after| < 1e-6`) и триггерить renumber колонки в TX.

### B2 — MAJOR: мультишаговые операции без транзакций
`.transaction(` встречается лишь в 3 репозиториях (project create+membership,
workspace). Каскадные удаления, перенос inbox→проект, архивация делегаций при move —
идут отдельными запросами. Крэш между шагами оставляет несогласованное состояние.

**Фикс:** обернуть в TX операции, меняющие 2+ таблицы (audit по use-case'ам:
`AssignTaskToProject`, delete-каскады, `SetProjectDispatcher`).

### B3 — MAJOR: почти нет FK → риск сирот
FK объявлены лишь в 6 миграциях из 98 (осознанно из-за errno 150 на MariaDB, см.
коммит про `site_artifacts`). Но отсутствие FK + отсутствие TX (B2) = осиротевшие
`task_comments`/`attachments`/`site_artifacts`/`live_sessions` при удалении родителя.

**Фикс:** для критичных связей либо вернуть FK с `ON DELETE CASCADE` (там, где errno
150 обходится единым charset/типом столбцов), либо явные каскадные удаления в TX +
фоновый sweeper-джоб, чистящий сирот по расписанию.

### B4 — MINOR: пагинация list-эндпоинтов
Проверить `ListTasks`/comments/activity/commits — отдают ли всё без limit/offset.
На больших проектах `taskRepository.list` тянет всю таблицу (это же бьёт по фронту,
см. A-ниже про `TaskDetailPage`). Ввести keyset-пагинацию на растущих таблицах.

### B5 — MINOR: перекрытие тиков планировщиков
`CommitSyncScheduler`/`DailyDigestScheduler` — `setInterval(60_000)` без `running`-флага.
Если `tick()` дольше минуты (медленный GitHub API) — следующий запустится поверх.
Сейчас смягчено `markCommitSyncRun`, но гонка на `enqueue` возможна.

**Фикс:** re-entrancy guard (`if (this.running) return; this.running = true; try {…} finally {…}`).

## 3. Фронтенд-архитектура

### A1 — CRITICAL: архитектурный линт мёртв
`client/eslint.config.js:54` — `boundaries/element-types` **никогда не срабатывает**:
нет `import/resolver`, все импорты для плагина «unknown», а `boundaries/no-unknown:'off'`
(стр. 106) их глотает. Экспериментально: `domain/`-файл с импортом
`../../infrastructure/http/httpClient` проходит lint зелёным. Реально защищает только
строковый `no-restricted-imports` на `@/infrastructure/{mock,http}/*` — его обходит
любой relative-импорт. CLAUDE.md обещает защиту слоёв, которой нет.

**Фикс:** `npm i -D eslint-import-resolver-typescript`; в settings
`'import/resolver': { typescript: { project: './tsconfig.app.json' } }`; выделить
`infrastructure/di` в отдельный element-type и явно разрешить `presentation → di`;
добавить в CI фикстуру-нарушение, ожидающую ошибку (ловить регрессию «правило умерло»).

### A2 — HIGH: нет ErrorBoundary
Греп `ErrorBoundary|componentDidCatch|errorElement` — 0. Любой throw в рендере →
React размонтирует всё дерево → **белый экран** без действий.

**Фикс:** корневой ErrorBoundary вокруг `RouterProvider` («Что-то сломалось —
перезагрузить»); `errorElement` на корневых роутах; отдельный boundary вокруг
`TaskDrawer` и kanban-области, чтобы крэш карточки не убивал доску.

### A3 — HIGH: нет глобальной обработки 401
`httpClient.ts` — 401 ловится только при стартовом `/me`. Истёкшая сессия посреди
работы → каждый запрос кидает `HttpError(401)`, компоненты глотают → UI молча мёртв
без редиректа.

**Фикс:** в `request()` при 401 диспатчить `window`-событие `pf:session-expired`;
`AuthProvider` слушает → `setStatus('anonymous')` → `ProtectedRoute` уводит на
`/login` с `state.from`. Заодно: `JSON.parse` в `httpClient.ts:33` без try/catch —
nginx-502 отдаёт HTML → `SyntaxError` вместо `HttpError(502)`; обернуть в try/catch.

### A4 — HIGH: монолитный бандл без code-splitting
`app/routes.tsx:2` — все 20 страниц статически; прод-сборка `index-*.js` = **1.95 MB
/ 560 KB gzip** одним куском. Анонимный посетитель публичной доски качает dnd-kit,
motion, admin, monitoring, finance, tiptap ради read-only канбана из 102 строк.

**Фикс:** `React.lazy` на каждую page + `Suspense` со скелетоном; для public-роутов
приоритетно (в идеале отдельный entry).

### A5 — HIGH: каскад ре-рендеров при resize сайдбара/панели
`useSidebarWidth.ts:98` — `setWidth` на каждый pointermove; provider-стек объявлен
**внутри** `AppShell` (`AppShell.tsx:170`), ни один provider-value не мемоизирован →
каждый pointermove ре-рендерит всех консьюмеров всех 16 контекстов.

**Фикс:** ширину писать в CSS-переменную через ref, коммитить в state на pointerup;
provider-стек вынести из layout-state-компонента; обернуть value в `useMemo`,
коллбеки в `useCallback`; для Projects/Workspaces разделить `ActionsCtx`/`DataCtx`.

### MINOR / MEDIUM (FE-арх)
- **a-m1** Ни одного `React.memo` в presentation → memo на `KanbanCard`/`KanbanColumn`/`ActivityItem`.
- **a-m2** Виртуализации нет нигде → заложить хотя бы для done-колонки и `LogTailViewer`.

- **a-m5** `useTasks.refetch` без `loadSeq`-гарда (в отличие от Projects) → SSE-шторм + optimistic move дают мигание старым состоянием.
- **a-m6** 3 независимые реализации SSE (notifications/chat/live) без backoff при фатальном разрыве → единый `useEventSource(url, handlers)` с экспон. backoff+jitter, reopen на `online`/`visibilitychange`.

## Системные выводы (общая картина)

1. **Права по умолчанию слишком широкие.** viewer читает секреты и может назначить
   себя диспетчером с доступом к чужому GitHub-токену. Нужен аудит всей матрицы
   `permissions.ts`: чтение кред/секретов и любые dispatcher-действия — не ниже
   `editor`/`owner`.
2. **Модель домена-в-одном-registrable.** Публичные доски и деплой-сайты на
   поддоменах `projectsflow.ru` ломают и безопасность (same-site CSRF, куки), и UX
   (сломанная регистрация, чужая палитра, нулевой SEO). Стратегически — вынести
   недоверенный/публичный контент на отдельный домен.
3. **Ошибка ≠ отказ ≠ пусто.** И фронт (`catch(()=>forbidden/notfound)`), и бэк
   (stale error-стейты) склеивают эти три состояния, показывая ложные сообщения без
   «Повторить». Нужна дисциплина: различать `HttpError.status`, всегда давать retry,
   сбрасывать error при успехе.
4. **Навигационное состояние живёт в storage, а не в URL.** Открытая задача, фильтры,
   возврат после логина — всё ad-hoc в session/localStorage; Back/share/deep-link
   ломаются. Инвертировать: навигация → URL, эфемерное (черновики, скролл) → storage.
5. **Отсутствие сетевого/крэш-ядра.** Нет ErrorBoundary, нет обработки 401, нет
   abort/timeout, `JSON.parse` без гарда, 3 копии SSE без backoff, мёртвый
   архитектурный линт. Один спринт на «ядро надёжности» (ErrorBoundary + 401-событие +
   `useEventSource` + резолвер eslint + code-splitting) даёт максимум прод-стабильности
   при минимуме кода.
6. **Mouse-first при PWA-позиционировании.** Клавиатура не работает в главном флоу,
   ключевая инфа и действия спрятаны за hover, таргеты < 44px. Нужен единый
   мобильный/a11y-аудит хит-зон и «hover — только ускоритель, не единственный путь».
