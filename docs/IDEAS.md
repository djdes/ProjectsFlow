## Диспетчер (ralph)

### Умный auto-sync проектов (вместо фиксированного интервала)
**Статус:** идея (сейчас — простой интервал 3 мин, `dispatch.ps1` → `$script:HourlySyncMinutes`).

Сейчас диспетчер раз в N минут делает ПОЛНЫЙ синк (список проектов + чтение KB-пути по
каждому + snapshot-детект + preflight) — это ~35–40 запросов за прогон. Мы снизили интервал
60 → 3 мин, чтобы новые self-serve проекты подхватывались быстро. По нагрузке это дёшево
(~0.2 req/s), но всё равно вхолостую гоняем полный синк, даже когда ничего не изменилось.

**Идея:** разделить лёгкую проверку и тяжёлый синк:
- каждые ~60с — ОДИН дешёвый запрос «список проектов, где я диспетчер» (только id+updated);
- сравнить с известным набором; **полный синк запускать ТОЛЬКО когда появился новый проект
  или изменился состав** (иначе — skip).

**Выгода:** новый проект подхватывается почти мгновенно (≤1 мин), запросов при этом МЕНЬШЕ,
чем при 3-мин полном синке (1 лёгкий/мин вместо ~40 каждые 3 мин), ноль работы вхолостую.
**Цена:** аккуратная правка цикла в `dispatch.ps1` (большой файл — делать осторожно) + рестарт.
