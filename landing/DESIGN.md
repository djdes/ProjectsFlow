# ProjectsFlow — лендинг · дизайн-гайдлайн (P0)

**Направление: «A · Dev-cool / технарь»** (выбрано пользователем 2026-06-30).
Аудитория — вайб-кодеры. Источник токенов — `src/styles/tokens.css`. Мокап-эталон —
`.design-variants/variant-a-dev-cool.html`.

## Вайб
Тёмная «терминальная» эстетика для разработчиков: глубокий near-black фон, электрический
**циан** как главный акцент, **сигнальный зелёный** для статусов, моно-шрифт в акцентных местах
(eyebrow, метки, псевдо-терминал, code-мотивы), тонкая координатная сетка и мягкие неон-свечения.
Спокойно по композиции, «дорого» по деталям — не «детский» неон.

## Палитра (из `tokens.css`)
| Назначение | Токен | Hex |
|---|---|---|
| Фон | `--bg` | `#06080d` |
| Поверхность / карточка | `--surface` / `--surface-2` | `#0c1018` / `#11161f` |
| Текст / приглушённый / тусклый | `--text`/`--muted`/`--dim` | `#e6edf3` / `#8b97a7` / `#5b6675` |
| Акцент (циан) | `--accent` / `--accent-deep` | `#22e0ff` / `#0892c0` |
| Текст на акценте | `--accent-ink` | `#04222b` |
| Сигнал (зелёный) / внимание | `--signal` / `--warn` | `#4ade80` / `#fbbf24` |
| Линии | `--line` / `--line-strong` | `rgba(255,255,255,.08/.14)` |

Градиенты: `--grad-accent` (кнопки/марки), `--grad-text` (акцентный текст заголовков).

## Типографика
- **Sans:** `--font-sans` (Inter Variable) — основной текст/заголовки.
- **Mono:** `--font-mono` — eyebrow, метки, терминал-визуал, числа в «командах». JetBrains Mono
  грузится при наличии; P1 МОЖЕТ добавить `@fontsource-variable/jetbrains-mono` (иначе системный моно).
- Шкала: `--fs-xs…--fs-3xl`, hero — `--fs-display`. Заголовки: `--tracking-tight`, `--lh-tight`.

## Компоненты / приёмы
- **Кнопки:** primary = `--grad-accent` + текст `--accent-ink` + лёгкое свечение; ghost = прозрачная
  + `--line`, hover усиливает рамку/фон. Радиус `--r-sm`/`--r`.
- **Карточки:** `--surface` + `--line`, hover — рамка → `--accent-line`, лёгкий подъём; радиус `--r`.
- **Eyebrow:** моно, циан, в «пилюле» с `--line`; рядом — зелёная точка-статус (`--signal`) со свечением.
- **Фон:** `.grid-bg` (координатная сетка) + мягкие cyan/green `.mesh-blob` свечения сверху/сбоку.
- **Терминал-визуал в Hero:** псевдо-окно с traffic-light кнопками и моно-командами
  (`pf task add … --col worker`) — как в мокапе A.
- **Pricing:** 3 карточки, Прайм выделен рамкой `--accent-line` + свечением; имена тарифов моно/циан.

## Анимации
- Easing — `--ease`; длительности `--dur-fast/--dur/--dur-slow`. Появление секций — gentle fade-up
  (как `[data-reveal]`). **Всё уважает `prefers-reduced-motion`** (включая three.js — см. P1).
- Без агрессивных бесконечных анимаций; неон «дышит» мягко.

## Do / Don't
- ✅ Циан — точечно (акценты/границы/свечения), не заливать им всё. Зелёный — только статусы.
- ✅ Моно — в технических/акцентных местах, не в длинных абзацах.
- ✅ Контраст текста ≥ AA на тёмном фоне.
- ❌ Не вводить второй случайный акцент; не смешивать со светлой темой.
- ❌ Не хардкодить цвета/размеры в секциях — только токены.

## Как применять (для P1)
- Импортируй ничего не нужно — `globals.css` уже тянет `tokens.css`. Используй `var(--…)` в
  scoped-стилях компонентов или Tailwind-классы, сопоставленные с токенами.
- Сверяйся с `.design-variants/variant-a-dev-cool.html` как с эталоном Hero/Features/Pricing.
- `globals.css`/`tokens.css`/`tailwind.config` — НЕ редактировать в P1 (владелец P0). Нужен новый
  токен — добавь сюда заметку и заведи переменную.
