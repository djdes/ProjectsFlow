# Accessibility audit — Notion AI chats

Исследование выполнено clean-room через видимый интерфейс в авторизованной вкладке Chrome на CDP `9222`. Это не формальная WCAG-сертификация: ниже только наблюдаемое поведение ролей, имён, фокуса и клавиатуры.

## Что сделано хорошо

- Почти все основные иконки имеют понятные accessible names: `New chat`, `Give context`, `Settings`, `Start voice recording`, `Submit AI message`, `Share chat`, `Start new chat`, `Pin chat`, `Minimize/Expand agent details`, `Delete, rename, and more…`.
- У действий ответа есть отдельные имена: `Copy response`, `Save to private pages`, положительная/отрицательная обратная связь, `Edit message`, `Copy text`.
- При клавиатурном фокусе composer и меню получают заметную синюю рамку.
- `Enter` подтверждает переименование; `Escape` закрывает меню, history popover и rename popover.
- Поле переименования имеет имя/placeholder `Rename chat` и заранее заполнено текущим заголовком.
- Кнопка панели деталей меняет accessible name вместе с состоянием: `Minimize agent details` ↔ `Expand agent details`.

## Проблемы, которые не стоит копировать

1. Кнопка видимого заголовка чата называется для assistive technology просто `history`. Лучше: `Открыть историю чатов: <заголовок>`.
2. Кнопка с лицом Notion AI в шапке сохранённого чата не получила наблюдаемого имени.
3. Чёрная кнопка перехода к последнему сообщению (32×32 над composer при прокрутке вверх) не получила наблюдаемого имени. Нужен `aria-label="Перейти к последнему сообщению"`.
4. Composer определяется по placeholder `Do anything with AI…`, отдельного `aria-label` не наблюдалось. Для ProjectsFlow нужен стабильный label независимо от placeholder.
5. При ширине `390px` и раскрытой боковой панели часть именованных кнопок физически находится за пределами viewport. Это уже не ARIA-дефект, а серьёзная проблема достижимости.
6. Строки сохранённых чатов представлены как `menuitem`, а не ссылки. Открытие в новой вкладке доступно через отдельное меню, но нативные ссылочные affordances отсутствуют.

## Требования для реализации в ProjectsFlow

- Реализовать полный roving/tab order: sidebar → заголовок/history → header actions → лента → composer.
- Для каждой иконки использовать стабильный `aria-label`, а tooltip считать только визуальным дополнением.
- После отправки оставлять фокус в composer; после переименования возвращать фокус на заголовок.
- В live-ленте использовать ненавязчивый `aria-live="polite"` для статуса выполнения и отдельный текстовый статус вместо одной мигающей точки.
- На mobile sidebar должен становиться modal drawer с focus trap, `Escape` и возвратом фокуса на кнопку открытия.
- У кнопки перехода вниз должно быть имя, клавиатурная активация и счётчик новых сообщений, если пользователь не внизу.
- Меню должно закрываться по `Escape`/outside click и восстанавливать фокус на инициаторе.
