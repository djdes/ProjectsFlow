# Сопоставление Preview с ProjectsFlow

## Что уже есть

- опубликованный домен результата и синяя плашка;
- внешний переход к результату;
- статусы приложения и app backend;
- GitHub onboarding;
- активность воркера и история изменений.

## Чего не хватает

- встроенного iframe результата;
- выбора безопасного пути внутри результата;
- desktop/tablet/mobile оболочки;
- отдельного верхнеуровневого Preview/Dashboard;
- обновления iframe и понятного состояния загрузки/ошибки;
- интегрированного health/security состояния публикации.

## Рекомендуемая реализация

1. Вынести Preview/Dashboard из Share popover в рабочую область проекта.
2. Разрешать только домен текущего проекта; поле должно менять path, а не произвольный origin.
3. Добавить desktop/tablet/mobile, reload и `Открыть отдельно`.
4. Показывать loading, timeout, HTTP/app error и последнюю успешную публикацию.
5. Позже добавить sitemap/page map. Полный Canvas и визуальный DOM-editor пока не переносить.
