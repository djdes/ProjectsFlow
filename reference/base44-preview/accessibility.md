# Доступность Preview

## Хорошо

- основные действия представлены нативными кнопками с читаемыми названиями;
- активные Preview/Dashboard имеют явное состояние;
- меню маршрутов и модальные окна доступны через фокусируемые элементы;
- адаптивные режимы не изменяют основной навигационный контекст.

## Риски

- часть иконок опирается на tooltip и визуальную форму;
- режим устройства показан иконкой, а не постоянной текстовой меткой;
- Canvas по природе требует альтернативы для клавиатуры и screen reader;
- приложение внутри iframe должно иметь собственную семантику и focus management;
- floating element toolbar должен оставаться достижимым с клавиатуры и не зависеть только от hover;
- selection overlay обязан иметь текстовое описание выбранного tag/role и live announcement;
- для destructive delete требуется подтверждение и возврат фокуса;
- code/source панели не должны вставлять HTML через `dangerouslySetInnerHTML`.

Для ProjectsFlow у каждой иконки нужны `aria-label`, tooltip и видимый focus; iframe должен иметь `title`, а смена маршрута — сообщаться через live region.
